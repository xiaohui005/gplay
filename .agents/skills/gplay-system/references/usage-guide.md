## 后续怎么慢慢形成标准

1. 第一次遇到问题：先完成任务，并记录到 `references/project-standards.md`。
2. 第二次遇到同类问题：补充更明确的规则。
3. 第三次仍然容易漏：抽成 `subskills/<name>/SKILL.md`。
4. 能自动检查的内容：写进 `scripts/`。
