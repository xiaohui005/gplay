## 开发默认约定

在项目结构明确前，执行任务时必须先发现：

1. 项目技术栈。
2. 前端和后端边界。
3. 依赖安装方式。
4. 启动、测试、打包命令。
5. 配置文件和环境变量位置。
6. 是否有已有编码、命名、路由、接口、权限和日志规范。

## 验收默认约定

项目命令明确前，验收顺序先按下面执行：

1. 针对改动文件做静态检查和编码检查。
2. 能发现测试命令时，运行相关测试。
3. 能发现前端打包命令时，运行前端打包。
4. 能发现后端重启命令时，后端相关改动后重启后端。
5. 如果命令缺失，交付时明确写出“未发现对应命令”，并把缺口记录在本文件。
