# 股票智能研判与买卖决策辅助系统 — 开发任务拆解清单

> 基于《股票智能研判与买卖决策辅助系统实施方案》v1.0，按 **后端、前端、数据采集、研判引擎、后台管理、测试** 六大方向拆解。  
> 每个任务包含：目标、输入、输出、接口、数据表、验收标准、优先级。

---

# 一、后端

## B-001 股票搜索接口

| 项 | 内容 |
|---|---|
| **任务目标** | 实现股票搜索 API，支持按股票代码、名称、拼音首字母搜索 A 股（沪/深/北交所），返回基础行情摘要 |
| **输入数据** | 关键词（代码/名称/拼音）、市场范围枚举 |
| **输出结果** | 匹配股票列表：symbol、name、market、latestPrice、changePercent、tradeStatus |
| **涉及接口** | `GET /api/stocks/search?keyword=` |
| **涉及数据表** | stock_basic、stock_quote_snapshot |
| **验收标准** | 输入"600000"返回浦发银行；输入"浦发"亦返回；输入"PF"拼音首字母亦返回；搜索结果含交易状态和最新价；退市/停牌股票仍可搜到但带状态标签 |
| **优先级** | P0 |

## B-002 个股详情与研判接口

| 项 | 内容 |
|---|---|
| **任务目标** | 实现个股研判聚合 API，聚合行情快照、评分、股票大师指导、条件清单、买卖计划、免责声明 |
| **输入数据** | symbol、market |
| **输出结果** | 见 20.1 接口字段字典：symbol、dataTime、delayMinutes、trendStatus、riskLevel、suggestion、score、masterGuidance、disclaimer |
| **涉及接口** | `GET /api/stocks/{symbol}/analysis` |
| **涉及数据表** | stock_basic、stock_quote_snapshot、stock_kline_daily、stock_risk_event、stock_analysis_result |
| **验收标准** | 正常股票返回完整研判结构；评分含趋势/量价/资金/板块/基本面/风险扣分；masterGuidance 含条件清单和买卖计划；数据时间、延迟分钟数必填 |
| **优先级** | P0 |

## B-003 预警管理接口

| 项 | 内容 |
|---|---|
| **任务目标** | 实现预警 CRUD，支持 PRICE_BREAKOUT / CHANGE_PERCENT / RISK_CHANGE / SUGGESTION_CHANGE 类型 |
| **输入数据** | userId、symbol、alertType、operator、targetValue、notifyChannels |
| **输出结果** | 预警对象含 alertId、userId、symbol、alertType、status、createdAt |
| **涉及接口** | `POST /api/stocks/{symbol}/alerts`、`GET /api/alerts`、`DELETE /api/alerts/{alertId}` |
| **涉及数据表** | stock_alert |
| **验收标准** | 创建预警返回完整对象；支持查看个人预警列表；预警触发后生成通知（接口留痕），推送文案不写"买入/卖出" |
| **优先级** | P0 |

## B-004 预警任务调度服务

| 项 | 内容 |
|---|---|
| **任务目标** | 实现预警检查定时任务，周期性扫描用户预警，匹配最新行情/风险/建议变化，触发后更新状态并生成通知 |
| **输入数据** | 启用状态的预警记录 + 最新行情快照 + 最新研判结果 |
| **输出结果** | 触发记录写入通知表；预警状态更新为 TRIGGERED |
| **涉及接口** | 无外部接口，内部服务调用 |
| **涉及数据表** | stock_alert、stock_quote_snapshot、stock_analysis_result |
| **验收标准** | 价格达到目标值后预警状态变更为 TRIGGERED；风险等级变化触发预警；建议变化触发预警；通知推送包含股票、触发条件和查看入口 |
| **优先级** | P0 |

## B-005 用户与权限服务

| 项 | 内容 |
|---|---|
| **任务目标** | 实现基础用户注册/登录和角色权限体系：游客、注册用户、付费用户、策略管理员、合规管理员、超级管理员、数据管理员 |
| **输入数据** | 用户信息、角色分配 |
| **输出结果** | 用户登录态、权限校验中间件、角色路由控制 |
| **涉及接口** | 登录、注册、角色分配接口 |
| **涉及数据表** | 用户表、角色表、权限表 |
| **验收标准** | 游客只能查看基础行情和部分研判；注册用户可设置预警；付费用户可看完整大师指导；管理员可访问后台接口；权限校验中间件拦截越权请求 |
| **优先级** | P0 |

## B-006 风控合规后端逻辑

| 项 | 内容 |
|---|---|
| **任务目标** | 实现停牌/ST/退市/重大利空强制拦截、数据缺失降级、敏感词校验中间件、日志留痕 |
| **输入数据** | 股票交易状态、风险标签、研判文案、用户输入 |
| **输出结果** | 拦截/降级决策；敏感词命中阻止发布；审计日志 |
| **涉及接口** | 研判 API 内部调用；文案发布前校验 |
| **涉及数据表** | stock_risk_event、audit_log |
| **验收标准** | 停牌股票不得输出交易建议；ST 股票强制高风险提示；数据缺失不输出强买入；敏感词命中阻止发布；拦截操作记录审计日志 |
| **优先级** | P0 |

| **输入数据** | 操作人ID、action、beforeValue、afterValue |

| **涉及接口** | 内部服务调用 |
| **涉及数据表** | audit_log |
| **验收标准** | 每次策略配置修改生成审计记录；数据采集手动操作记录审计；审计日志支持按操作人、时间、操作类型查询 |
| **优先级** | P0 |

## B-008 数据模型与数据库初始化

| 项 | 内容 |
|---|---|
| **任务目标** | 根据 13 节建议数据模型，设计并创建全部数据库表/集合，含索引、枚举约束、字段注释 |
| **输入数据** | stock_basic、stock_quote_snapshot、stock_kline_daily、stock_risk_event、stock_analysis_result、stock_alert、strategy_config、audit_log、data_collection_job、data_collection_error、raw_market_data |
| **输出结果** | 数据库建表脚本、迁移文件 |
| **涉及接口** | 无 |
| **涉及数据表** | 全部上述表 |
| **验收标准** | 表结构完整、主键/外键/索引正确；枚举字段与文档一致；raw_market_data 含 payload JSON 字段；带 batchId、dataVersion 串联字段 |
| **优先级** | P0 |

## B-009 异常降级与错误码体系

| 项 | 内容 |
|---|---|
| **任务目标** | 统一错误码定义、异常降级处理：行情不可用、行情延迟、采集失败、模型失败、信号冲突等场景 |
| **输入数据** | 各服务调用结果、数据状态 |
| **输出结果** | 降级后的响应（含错误码、降级提示） |
| **涉及接口** | 全部业务 API |
| **涉及数据表** | 无 |
| **验收标准** | 行情不可用时返回"当前行情数据暂不可用"；信号冲突时输出观望；模型失败时规则兜底；采集任务失败保留上一可用数据并标记延迟 |
| **优先级** | P0 |

# 三、数据采集

## D-001 数据源配置模块

| 项 | 内容 |
|---|---|
| **任务目标** | 实现数据源配置管理：供应商名称、鉴权信息、限频、启停状态、支持的数据类型、API 端点 |
| **输入数据** | 数据源参数（source_code、授权信息、base_url、rate_limit、数据类型映射） |
| **输出结果** | 数据源配置持久化，采集任务读取配置执行 |
| **涉及接口** | `POST /api/admin/data-sources`、`GET /api/admin/data-sources` |
| **涉及数据表** | data_source_config（新增）或 strategy_config 扩展 |
| **验收标准** | 配置可增删改查；采集任务运行时读取启用的数据源配置；限频配置生效；修改配置后不中断正在执行的任务 |
| **优先级** | P0 |

## D-002 股票基础信息采集

| 项 | 内容 |
|---|---|
| **任务目标** | 定时采集 A 股全市场股票基础信息：代码、名称、市场、交易状态；支持盘前每日 1 次同步 |
| **输入数据** | 数据源 API（Tushare/BaoStock/AKShare 等） |
| **输出结果** | 写入 stock_basic 表 |
| **涉及接口** | 内部采集任务 |
| **涉及数据表** | stock_basic、raw_market_data、data_collection_job、data_collection_error |
| **验收标准** | 成功写入股票代码、名称、市场、状态；更新已存在记录；失败记录错误日志并重试 |
| **优先级** | P0 |

## D-003 交易状态同步

| 项 | 内容 |
|---|---|
